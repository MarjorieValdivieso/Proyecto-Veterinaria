# 🐾 Sistema de Gestión Veterinaria

## Descripción

Este proyecto consiste en el diseño e implementación de una base de datos para una clínica veterinaria utilizando **MySQL**. El sistema permite gestionar la información de clientes, mascotas, veterinarios, citas, consultas, tratamientos, vacunas, inventario, proveedores, compras y ventas, garantizando la integridad, seguridad y disponibilidad de los datos.

## Objetivos

- Diseñar una base de datos normalizada para una clínica veterinaria.
- Administrar la información clínica y administrativa de forma eficiente.
- Implementar mecanismos de seguridad mediante usuarios, roles y permisos.
- Automatizar procesos utilizando procedimientos almacenados, funciones y triggers.
- Garantizar la consistencia de la información mediante transacciones.

## Funcionalidades

- Modelo Entidad-Relación (MER).
- Modelo Lógico de la base de datos.
- Normalización hasta la Tercera Forma Normal (3FN).
- Creación de tablas con claves primarias y foráneas.
- Inserción de datos de prueba.
- Consultas SQL simples y avanzadas.
- Vistas para consultas de información.
- Funciones y procedimientos almacenados.
- Triggers para automatización de procesos.
- Transacciones (COMMIT y ROLLBACK).
- Gestión de usuarios, roles y permisos.
- Respaldo y restauración de la base de datos.
